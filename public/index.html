How this frontend works

The browser only keeps an HttpOnly session cookie. The backend stores the OAuth tokens and exposes a small BFF surface through /web/me, /web/apps, and /web/logout.

Login

Starts a server-managed login flow and keeps tokens on the backend instead of in browser JavaScript.

Register app

Shows client ID by default and only displays a client secret if the backend actually returns one.
